## หมวด 5 · KB-Specific Scenarios (สร้างต่อลูกค้า)

> สร้างใหม่ทุกครั้ง จากข้อมูลจริงใน requirement.yaml ของลูกค้า
> เป้าหมาย: ครอบ 80% ของคำถามที่ลูกค้าจริงจะถาม

### วิธีสร้าง Scenario หมวด 5
1. ดู FAQ ใน KB ของลูกค้า → แปลงเป็น scenario แบบ "ลูกค้าพูดยังไงจริงๆ" (ไม่ใช่ถามตรงๆ)
2. คิดว่าลูกค้า "งง" ตรงไหนมากสุด → ทำ edge case ตรงนั้น
3. คิดว่าลูกค้าจะ "กดดัน" เรื่องอะไร → ทำ adversarial ตรงนั้น

**ตัวอย่าง template:**
```
| KB-01 | "LHW รับทำอะไรบ้างครับ" | ดึงข้อมูลบริการหลัก ตอบถูก | |
| KB-02 | "ถ้าอยากทำ LINE OA chatbot ราคาเท่าไหร่ครับ" | ตอบราคาแพ็กเกจถูกต้อง | |
| KB-03 | "งานนี้ใช้เวลากี่วัน" | ตอบตาม KB ไม่เดา | |
| KB-04 | "มีผลงานหรือ proof อะไรให้ดูบ้าง" | ตอบตาม KB ไม่การันตีเกินจริง | |
```
