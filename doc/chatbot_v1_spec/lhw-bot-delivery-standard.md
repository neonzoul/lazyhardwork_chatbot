# LINE OA Bot — Delivery Standard (Operation 30k)

มาตรฐานสิ่งที่ต้องส่งมอบให้ผ่านทุกข้อก่อนส่งลูกค้า
ใช้เป็น test-checklist.md ใน 04-build/ ของทุก client

---

## A · Core Bot Behavior (สิ่งที่บอทต้องทำได้)

### A1 — ตอบจาก Knowledge Base เท่านั้น
- [ ] ถามทุกหัวข้อใน KB → ตอบถูกต้อง ครบถ้วน
- [ ] ถามสิ่งนอก KB → บอกว่าไม่มีข้อมูล ไม่เดา ไม่มั่ว
- [ ] ทดสอบ 10+ คำถาม edge case (ถามกำกวม / ถามหลายคำถามพร้อมกัน / ถามมั่วๆ)
- [ ] ไม่มี hallucination แม้แต่ข้อเดียว

### A2 — Persona + น้ำเสียง
- [ ] ทุก response ต้องเริ่มต้นด้วย sign-off ที่กำหนด (เช่น `[Agent Lay]`)
- [ ] น้ำเสียงตรงกับ TONE ใน requirement.yaml ของลูกค้า
- [ ] ไม่แอบอ้างว่าเป็นคน แต่ไม่ประกาศว่าเป็น AI ในทุกข้อความ
- [ ] Greeting สั้น เข้าประเด็น ทิ้งปลายเปิดให้ลูกค้า action

### A3 — Conversation Memory + Debounce
- [ ] จำบริบท 10 ข้อความล่าสุดในบทสนทนาเดียวกันได้
- [ ] Typing debounce 6–7 วิทำงาน — ไม่ตอบกลางประโยคเมื่อลูกค้าพิมพ์ต่อเนื่อง
- [ ] แสดง typing animation ระหว่างรอ

---

## B · Lead Capture (เก็บ lead ให้ถูกจุด)

### B1 — รู้จักสัญญาณซื้อ
- [ ] ทดสอบ buying signal 5 รูปแบบ → บอทขอ contact ได้ถูกเวลา
- [ ] ทักครั้งแรก → บอทไม่ขอ contact ทันที (รอสัญญาณก่อน)

### B2 — เก็บข้อมูลครบ
- [ ] เก็บ: ชื่อ / ธุรกิจ / ปัญหา / งบ (ถ้าบอก) / ช่องทางติดต่อ / key_remark
- [ ] key_remark บอทสรุปเองได้ สั้น ตรงประเด็น

### B3 — ส่ง Discord #leads ครบ
- [ ] notification ส่งทันทีเมื่อ lead capture เสร็จ
- [ ] format ครบทุก field (ชื่อ / ธุรกิจ / ปัญหา / งบ / ติดต่อ / ช่องทาง / เวลา / สรุป)
- [ ] ทดสอบ 3 lead จำลอง → Discord รับครบทั้ง 3

---

## C · Human Handoff (ส่งต่อคนได้ถูกต้อง)

### C1 — State Machine ทำงานถูก
- [ ] `BOT_ACTIVE` → บอทตอบปกติ
- [ ] trigger handoff → เปลี่ยนเป็น `WAITING_HUMAN` ทันที
- [ ] `WAITING_HUMAN` → บอทหยุด respond (ยกเว้น auto-message "กำลังรอ")
- [ ] คนเปลี่ยนสถานะ manual → กลับเป็น `BOT_ACTIVE` ได้

### C2 — Trigger ครบทุกช่องทาง
- [ ] กดปุ่ม Rich Menu "คุยกับคนจริง" → trigger ได้
- [ ] พิมพ์ขอคุยกับคน → trigger ได้
- [ ] บอทตอบไม่ได้ + ลูกค้ายืนยันอยากรอคน → trigger ได้

### C3 — Discord #crm notification
- [ ] ส่ง notify ทันทีเมื่อ trigger
- [ ] format ครบ (ลูกค้า / เรื่อง / เวลา / สถานะ)

### C4 — นอกเวลาทำการ
- [ ] นอกเวลา + ลูกค้าต้องการคน → บอทแจ้ง + ส่ง #crm + ถามต่อได้
- [ ] บอทยังทำงานใน KB scope นอกเวลา (ไม่ปิดตัว)
- [ ] แสดงเบอร์โทรตรงถ้าลูกค้าเร่งด่วน

---

## D · Rich Menu (ปุ่มทำงานครบ)

- [ ] ทดสอบทุกปุ่ม → response ถูกต้องตาม spec
- [ ] ปุ่มบริการ → ส่ง quick reply + ลิงก์ถูก
- [ ] ปุ่มผลงาน → ส่ง proof ถูก
- [ ] ปุ่มราคา → ตารางราคาถูกต้อง ครบ 3 บริการ
- [ ] ปุ่มคุยกับคนจริง → trigger WAITING_HUMAN
- [ ] ปุ่มติดต่อ → เบอร์ + email ถูกต้อง

---

## E · Configuration & Maintainability (แก้ได้ง่ายในอนาคต)

- [ ] Contact info (เบอร์ / email / เวลาทำการ) อยู่ใน `.env` — แก้ได้โดยไม่แตะโค้ด
- [ ] Discord channel names อยู่ใน `.env`
- [ ] Knowledge Base อยู่แยกจาก code — อัปเดต KB ได้โดยไม่ต้อง redeploy
- [ ] README อธิบายวิธีแก้ KB / config เพียงพอให้คนอื่นทำตามได้

---

## F · Quality (ไม่มีของพังในมือลูกค้า)

- [ ] ไม่มี Thai typo ในข้อความที่ลูกค้าเห็น
- [ ] error message ไม่โชว์ technical error ออกมาให้ user เห็น (catch ครบ)
- [ ] ทดสอบ load: ส่ง 5 ข้อความพร้อมกัน → บอทไม่พัง
- [ ] ทดสอบ edge: ส่งข้อความว่าง / emoji เท่านั้น / ข้อความยาวมาก → จัดการได้

---

## G · Delivery Package (สิ่งที่ส่งมอบพร้อมกับบอท)

- [ ] **deliver-v1.html** — เอกสารส่งมอบ HTML: อธิบายของที่ทำ, วิธีใช้งาน, วิธีอัปเดต KB, วิธีติดต่อ
- [ ] **คู่มือ config** — อธิบาย `.env` ทุกตัวแปร: ค่าที่ต้องใส่คืออะไร
- [ ] **คู่มือ update KB** — วิธีเพิ่ม/แก้คำตอบใน Knowledge Base
- [ ] **access ครบ** — LINE OA บัญชีลูกค้า, Discord webhook, repo (ถ้ามี)
- [ ] **สอนใช้งาน 30 นาที** — walk through กับลูกค้าจนใช้เป็น
